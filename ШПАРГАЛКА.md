### ❌ ПРОБЛЕМА: В профиле "Сыр = 0" (вместо реального баланса)

**Причина:** Telegram WebView или GitHub CDN закэшировали старый JS файл.

**Как проверить:**
1. Открой в браузере: `https://mir-tankov-production.up.railway.app/api/me?telegram_id=ВАШ_ID`
2. Если `cheese` не 0 — проблема в кэше фронтенда, не в бэкенде

**Как починить (быстро):**  
В `webapp/profile.html` уже есть ИНЛАЙН-скрипт `loadCheeseInline()` в конце файла.  
Он грузит сыр напрямую из API при каждом открытии страницы — **независимо от кэша JS-файлов**.

Если вдруг сломается снова:  
```html
<!-- Вставить в конец profile.html перед </body> -->
<script>
(function() {
    var API = 'https://mir-tankov-production.up.railway.app';
    function getId() {
        var tg = window.Telegram && window.Telegram.WebApp;
        if (tg && tg.initDataUnsafe && tg.initDataUnsafe.user) return String(tg.initDataUnsafe.user.id);
        var urlId = new URLSearchParams(window.location.search).get('telegram_id');
        if (urlId) return urlId;
        return localStorage.getItem('my_telegram_id');
    }
    async function load() {
        var id = getId(); if (!id) return;
        var r = await fetch(API + '/api/profile/battle-stats?telegram_id=' + id);
        var s = await r.json();
        var el = document.getElementById('progCoins');
        if (el && s.cheese_balance !== undefined) el.textContent = s.cheese_balance.toLocaleString();
    }
    document.readyState === 'loading' ? document.addEventListener('DOMContentLoaded', load) : load();
})();
</script>
```

### ❌ ПРОБЛЕМА: WebApp не обновляется в Telegram (старая версия)

**Причина:** Telegram кэширует WebApp-страницы.

**Как починить:**
1. В боте (`bot.py`, строка ~267) бот уже добавляет `?_t=timestamp` к ссылке — это главный anti-cache
2. Убедись что `?_t=` есть в строке:  
   ```python
   user_webapp_url = f"{WEBAPP_URL}?_t={cache_bust}&telegram_id={message.from_user.id}"
   ```
3. В HTML файлах скрипты должны иметь версию: `<script src="js/app.js?v=57">`  
   При изменениях — увеличить цифру (`v=58`, `v=59` и т.д.)
4. Юзер: закрой WebApp свайпом вниз → подожди 30 сек → открой снова

### ❌ ПРОБЛЕМА: Статистика сражений (1vs1/GC/Team) показывает 0

**API для проверки:**
```
GET /api/profile/battle-stats?telegram_id=XXX
```
Должно вернуть:
```json
{
  "cheese_balance": 3277,
  "battles": {
    "total": 2,
    "pvp": {"total": 2, "wins": 0, "losses": 2}
  }
}
```

**Что считается "сыгранным":**  
Только `arena_challenges` со статусом `'finished'`.  
Если бой в статусе `'active'` или `'pending'` — он идёт сейчас, в историю не попадает.

**Функция в bot.py:** `api_profile_battle_stats` (~строка 7339)

### ❌ ПРОБЛЕМА: navigation — переход на профиль не работает / теряет telegram_id

**Причина:** Некоторые страницы использовали `location.href='profile.html'` без передачи ID.

**Правильный способ** — через функцию `goToProfile()` (добавлена в `app.js`):
```html
<!-- Было (неправильно) -->
<div onclick="location.href='profile.html'">Профиль</div>

<!-- Стало (правильно) -->
<div onclick="goToProfile()">Профиль</div>
```
`goToProfile()` автоматически добавляет `telegram_id` и `_t=timestamp` в URL.

**Если нужно добавить в новую страницу** — просто использовать `goToProfile()`.  
Функция определена в `webapp/js/app.js` в самом конце файла.

---

### ❌ ПРОБЛЕМА: Railway не задеплоил новый код

**Симптом:** Изменения в `bot.py` не применились.

**Проверить логи Railway:**  
Dashboard → Твой проект → Deployments → последний deploy → смотри ошибки

**Важно:** Railway деплоит автоматически при `git push origin main`.  
Файлы `webapp/` и `site/` — это **GitHub Pages**, Railway их не видит.  
`bot.py`, `database.py`, `requirements.txt` — это Railway.

---

### ❌ ПРОБЛЕМА: База данных на Railway пустая / другая

**Важно:** `ecosystem.db` НЕ хранится в git (в `.gitignore`).  
На Railway есть свой экземпляр БД в файловой системе контейнера.  
**Локальная БД и Railway БД — это разные файлы!**

Данные в Railway — живые данные пользователей (правильные).  
Локальная `ecosystem.db` — для тестов.
