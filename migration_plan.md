# 🚀 План переезда на российский хостинг (Amvera / Timeweb)

Для того чтобы всё работало так же автоматически, как на Railway, следуй этому чек-листу.

## 🛠 Шаг 0: Подготовка (на стороне Railway)
Перед тем как всё выключить, сохрани данные:
1.  **Environment Variables:** Зайди в проект на Railway -> Settings -> Variables. Скопируй все ключи (BOT_TOKEN, API_KEY, DATABASE_URL и т.д.) в текстовый файл.
2.  **Database:** Если база данных встроенная в Railway, нужно будет сделать экспорт (дамп) или просто начать новую (если это не критично).

---

## 🏎 Вариант А: Amvera Cloud (Рекомендуемый)

1.  **Регистрация:** Зайди на [amvera.ru](https://amvera.ru/) и создай аккаунт.
2.  **Создание проекта:** Нажми «Создать проект».
3.  **Подключение:** Выбери способ «GitHub». Авторизуй Amvera и выбери свой репозиторий `Mir-Tankov`.
4.  **Конфигурация:**
    *   Тип сервиса: **Docker** (так как я уже создал для тебя `Dockerfile`).
    *   Amvera сама увидит файл и предложит настройки.
5.  **Переменные:** В разделе «Переменные окружения» (Environment) вставь все ключи из Шага 0.
6.  **Деплой:** Нажми «Запустить». Теперь при каждом моем пуше в GitHub, Amvera будет сама обновлять бота.

---

## 🌐 Вариант Б: Timeweb Cloud (Надежный / Pro)

1.  **Регистрация:** Зайди на [timeweb.cloud](https://timeweb.cloud/).
2.  **Облачные приложения:** В левом меню выбери **«App Service»** (Облачные приложения).
3.  **Создание:** Нажми «Создать». Выбери «GitHub».
4.  **Настройка:**
    *   Выбери репозиторий.
    *   Тип приложения: **Docker**.
    *   Ресурс: Выбери минимальный подходящий (обычно 1-2 ГБ ОЗУ хватает с запасом).
5.  **Переменные:** В поле «Environment variables» добавь свои ключи.
6.  **Запуск:** Готово! Бот получит домен или будет работать просто по API.

---

## 🔗 Шаг 3: Обновление Веб-сайта
Когда бэкенд запустится на новом месте:
1.  У тебя появится новый адрес API (например, `iserveri-bot.amvera.io`).
2.  Мне нужно будет просто заменить одну строчку в файле `website/js/api.js`, чтобы сайт начал обращаться к новому серверу в РФ.

### Почему это сработает?
*   **Dockerfile** универсален. Он работает везде одинаково.
*   **GitHub** остается посредником. Я пушу туда — сервер подхватывает.
